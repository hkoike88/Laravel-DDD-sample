# Initiatives - イニシアチブ管理

最終更新: 2024-04-22

## 概要

このディレクトリは **Initiative（イニシアチブ）** を管理する。
イニシアチブは組織の戦略的な事業目標を達成するための取り組み。

---

## 階層構造

```
Initiative（事業ゴール）
    └─ Use Case（業務行動）
         └─ Epic
             └─ User Story
                 ├─ Design Tasks   ← 外部設計
                 └─ Spec Tasks     ← 詳細設計
```

| 階層 | 説明 | 管理場所 |
|------|------|---------|
| Initiative | 事業ゴール（なぜやるか） | `01_vision/initiatives/` |
| Use Case | 業務行動（誰が何をするか） | `01_vision/initiatives/{ID}/usecases/` |
| Epic | 大きな機能単位 | `02_backlog/epics/` |
| User Story | ユーザー価値単位 | `02_backlog/epics/{EP-ID}/stories/` |
| Tasks | 作業単位（Design/Spec） | 各Storyに紐づく |

### Design Tasks と Spec Tasks

| 区分 | 内容 | 例 |
|------|------|-----|
| **Design（外部設計）** | ユーザー・外部から見える部分 | 画面設計、API I/F、入出力定義 |
| **Spec（詳細設計）** | 内部実装の詳細 | DB設計、処理ロジック、クラス設計 |

## イニシアチブ概要

### INI-000: 開発環境構築

Docker Compose を使用した開発環境の構築。フロントエンド・バックエンド・データベースが連携動作する状態を実現。

### INI-001: 認証・利用者管理基盤

職員がシステムを安全に利用できる認証基盤と、一般利用者向け公開機能（蔵書検索）を構築。INI-002（MVP）の前提条件となる基盤を整備。

**含まれる Use Case**:
- UC-001-001 職員ログイン
- UC-001-002 職員ダッシュボード表示
- UC-001-003 職員アカウント作成
- UC-001-004 職員アカウント編集
- UC-001-005 職員アカウント無効化
- UC-001-006 蔵書検索（公開機能）
- UC-001-007 職員ログアウト
- UC-001-008 セキュリティ対策準備

※ 一般利用者向けログイン・マイページは Phase 3 以降で検討

### INI-002: 図書館業務デジタル化プロジェクト（MVP）

Excel 中心の運用からデジタル化されたシステムへの移行。蔵書登録・貸出返却・予約管理・利用者管理の基本機能を構築。

**含まれる Use Case**:
- UC-002-001 蔵書登録
- UC-002-002 貸出処理
- UC-002-003 返却処理
- UC-002-004 予約登録
- UC-002-005 予約管理
- UC-002-006 利用者登録
- UC-002-007 利用者検索

※ 蔵書検索（公開機能）は INI-001 で提供

### INI-003: 機能拡張

運用効率向上とデータ活用の開始。統計レポート機能、遅延通知メール、ISBN API連携などの機能拡張。

**含まれる Use Case**:
- UC-003-001 統計レポート生成
- UC-003-002 遅延通知送信
- UC-003-003 ISBN自動取得

### INI-004: スマートフォン対応

利用者の利便性向上とアクセシビリティ拡大。レスポンシブUI、マイページ機能、プッシュ通知を提供。

**含まれる Use Case**:
- UC-004-001 マイページ閲覧
- UC-004-002 オンライン予約
- UC-004-003 プッシュ通知受信

### INI-005: セルフ貸出・ICタグ対応

完全デジタル化と無人運用の基盤構築。セルフ貸出端末、ICタグ管理、学校図書館連携を実現。

**含まれる Use Case**:
- UC-005-001 セルフ貸出
- UC-005-002 セルフ返却
- UC-005-003 蔵書棚卸し
- UC-005-004 学校蔵書検索

## Use Case から Epic への展開

Use Case を洗い出した後、Epic・Story に分解してバックログで管理する。

```
01_vision/initiatives/INI-001/usecases/UC-001-001
                    │
                    ▼
02_backlog/epics/EP-001/
    ├── epic.md
    └── stories/
        ├── ST-001.md
        │   ├─ Design Tasks
        │   └─ Spec Tasks
        └── ST-002.md
```
