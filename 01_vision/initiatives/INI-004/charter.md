# INI-004: スマートフォン対応イニシアチブ（Phase 3）

最終更新: 2024-04-22

---

## 概要

本イニシアチブは、**利用者の利便性向上とアクセシビリティ拡大** を目的とする。
スマートフォン・タブレットからの蔵書検索、マイページ機能の提供、プッシュ通知による情報提供を実現する。

---

## ビジネス目標

| 項目 | 内容 |
|------|------|
| **目的** | 利用者がいつでもどこでも図書館サービスにアクセスできる環境の構築 |
| **成功指標** | スマートフォンからの蔵書検索利用率30%、利用者満足度4.0以上 |
| **対象者** | 図書館利用者（特に若年層・働き世代） |
| **期待効果** | 来館前の事前検索、予約状況確認の利便性向上 |

---

## スコープ

### In Scope（対象範囲）

| カテゴリ | 内容 | 優先度 |
|----------|------|--------|
| **レスポンシブUI** | スマートフォン・タブレット対応のUI設計 | Must |
| **マイページ** | 貸出履歴、予約状況確認機能 | Must |
| **プッシュ通知** | 予約可能通知、返却リマインダー | Should |
| **オンライン予約** | 利用者自身による予約登録 | Should |

### Out of Scope（対象外）

| カテゴリ | 理由 |
|----------|------|
| ネイティブアプリ | PWAで対応（開発コスト削減） |
| セルフ貸出 | Phase 4（INI-005）で対応 |
| 電子書籍 | 将来検討 |

---

## 前提条件

- INI-002（MVP）、INI-003（機能拡張）が完了し、安定稼働していること
- 利用者認証機能が実装されていること
- HTTPS環境が整備されていること

---

## 主要機能

### 1. レスポンシブUI

| 機能 | 説明 |
|------|------|
| モバイルファースト設計 | スマートフォンでの操作性を最優先 |
| タッチ操作最適化 | ボタンサイズ、スワイプ操作対応 |
| 高齢者配慮 | フォントサイズ調整、コントラスト確保 |

### 2. マイページ機能

| 機能 | 説明 |
|------|------|
| 貸出状況確認 | 現在借りている本の一覧と返却期限 |
| 貸出履歴 | 過去の貸出履歴の確認 |
| 予約状況確認 | 予約中の本と待ち順番 |
| お気に入り登録 | 気になる本の保存 |

### 3. プッシュ通知

| 機能 | 説明 |
|------|------|
| 予約可能通知 | 予約していた本が利用可能になった時 |
| 返却リマインダー | 返却期限の3日前 |
| 新着本通知 | お気に入りジャンルの新着本情報 |
