# INI-000: 開発環境構築イニシアチブ

最終更新: 2024-04-22

---

## 概要

本イニシアチブは、図書館デジタル化システムの開発を開始するための **開発環境構築** を目的とする。
Docker Compose を使用したコンテナベースの開発環境を整備し、フロントエンド・バックエンド・データベースが連携動作する状態を構築する。

---

## ビジネス目標

| 項目 | 内容 |
|------|------|
| **目的** | 開発チームが即座に開発を開始できる環境を整備する |
| **成功指標** | 全サービスが起動し、フロントエンドからバックエンドAPI経由でDBアクセスができる |
| **対象者** | 開発チームメンバー |
| **期待効果** | 環境差異によるトラブル削減、オンボーディング時間の短縮 |

---

## スコープ

### In Scope（対象範囲）

| カテゴリ | 内容 |
|----------|------|
| **Docker環境** | Docker Compose による開発環境構築 |
| **バックエンド** | Laravel 11.x プロジェクト初期設定 |
| **フロントエンド** | React + TypeScript + Vite プロジェクト初期設定 |
| **データベース** | MySQL 8.0 コンテナ設定 |
| **リバースプロキシ** | Nginx 設定 |
| **開発ツール** | phpMyAdmin 設定 |
| **認証基盤** | 職員認証機能（ログイン/ログアウト）、Laravel Sanctum によるセッション認証 |

### Out of Scope（対象外）

| カテゴリ | 理由 |
|----------|------|
| 本番環境構築 | 別イニシアチブで対応 |
| CI/CD 設定 | 開発が進んでから対応 |
| 機能実装 | INI-001 以降で対応 |

## 完了条件

1. `docker compose up` で全サービスが起動すること
2. フロントエンド（http://localhost:5173）にアクセスできること
3. バックエンドAPI（http://localhost:8000/api）にアクセスできること
4. phpMyAdmin（http://localhost:8080）でDBに接続できること
5. フロントエンドからバックエンドAPIを呼び出し、DBのデータを取得できること
6. 職員がログイン/ログアウトできること
7. 未ログイン状態では職員向け機能にアクセスできないこと
