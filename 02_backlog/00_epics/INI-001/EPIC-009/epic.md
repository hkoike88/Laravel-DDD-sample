# EPIC-009: 利用者アカウント編集機能

最終更新: 2025-12-26

---

## 概要

図書館職員が既存利用者の情報を編集する機能を実装する。住所変更、連絡先変更、有効期限の更新などを行い、変更履歴を記録する。

## ビジネス価値

利用者情報を最新の状態に保ち、連絡先変更や住所変更に対応する。有効期限の更新により継続利用を可能にし、変更履歴の記録により監査対応を実現する。

---

## 受け入れ条件

1. 職員が利用者情報を編集できること
2. 利用者番号は変更不可であること
3. 有効期限を更新（延長）できること
4. 変更履歴が記録されること
5. 編集操作が監査ログに記録されること
6. 個人情報が暗号化して保存されること

## 技術仕様

### API エンドポイント

| メソッド | エンドポイント | 説明 | 認証 | 権限 |
|----------|---------------|------|------|------|
| GET | `/api/patrons/{id}` | 利用者詳細取得 | 必須 | 職員 |
| PUT | `/api/patrons/{id}` | 利用者情報更新 | 必須 | 職員 |
| POST | `/api/patrons/{id}/renew` | 有効期限更新 | 必須 | 職員 |

### リクエスト/レスポンス

#### PUT /api/patrons/{id}

**リクエスト:**
```json
{
  "name": "山田 太郎",
  "nameKana": "やまだ たろう",
  "birthDate": "1990-05-15",
  "address": "〒123-4567 東京都○○区...",
  "phoneNumber": "090-1234-5678",
  "patronType": "general",
  "notes": "住所変更（2025/12/26）"
}
```

**成功レスポンス (200):**
```json
{
  "message": "利用者情報を更新しました",
  "patron": {
    "id": "01HV...",
    "patronNumber": "P2025000001",
    "name": "山田 太郎",
    "updatedAt": "2025-12-26T10:00:00+09:00"
  }
}
```

#### POST /api/patrons/{id}/renew

**成功レスポンス (200):**
```json
{
  "message": "有効期限を更新しました",
  "patron": {
    "id": "01HV...",
    "patronNumber": "P2025000001",
    "expiresAt": "2026-12-26",
    "renewedAt": "2025-12-26T10:00:00+09:00"
  }
}
```

## 非機能要件

| 項目 | 要件 |
|------|------|
| パフォーマンス | 更新処理は2秒以内に完了する |
| セキュリティ | 個人情報は暗号化して保存 |
| 監査 | 更新操作と変更内容を audit チャンネルに記録 |
| データ保持 | 変更履歴は5年間保存する |

|----------|-----------|
