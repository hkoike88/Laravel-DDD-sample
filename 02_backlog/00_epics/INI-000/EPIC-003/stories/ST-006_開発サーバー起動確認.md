# ST-006: 開発サーバーの起動確認

最終更新: 2025-12-23

---

## ストーリー

**開発者として**、開発サーバーが正常に起動することを確認したい。
**なぜなら**、すべての設定が正しく完了し、開発を開始できる状態であることを保証したいからだ。

## 受け入れ条件

1. [ ] `npm run dev` で開発サーバーが起動すること
2. [ ] http://localhost:5173 でアプリケーションが表示されること
3. [ ] TypeScript の型チェックが通ること（`npx tsc --noEmit`）
4. [ ] ESLint エラーがないこと（`npm run lint`）
5. [ ] `npm run build` でビルドが成功すること
6. [ ] ホットリロードが正常に動作すること

---

## 技術仕様

### 確認手順

#### 1. 開発サーバー起動

```bash
# フロントエンドコンテナに入る
docker compose exec frontend bash

# 開発サーバー起動
npm run dev

# 出力例:
#   VITE v6.x.x  ready in xxx ms
#   ➜  Local:   http://localhost:5173/
#   ➜  Network: http://0.0.0.0:5173/
```

#### 2. ブラウザ確認

- http://localhost:5173 にアクセス
- アプリケーションが正常に表示されることを確認
- Tailwind CSS のスタイルが適用されていることを確認

#### 3. 型チェック

```bash
npx tsc --noEmit

# エラーがなければ成功
```

#### 4. リントチェック

```bash
npm run lint

# エラーがなければ成功
```

#### 5. ビルド確認

```bash
npm run build

# 出力例:
# vite v6.x.x building for production...
# ✓ xxx modules transformed.
# dist/index.html                   x.xx kB │ gzip: x.xx kB
# dist/assets/index-xxxx.css        x.xx kB │ gzip: x.xx kB
# dist/assets/index-xxxx.js       xxx.xx kB │ gzip: xx.xx kB
# ✓ built in xxxms
```

#### 6. ホットリロード確認

```bash
# 開発サーバー起動中に
# src/app/App.tsx を編集して保存
# ブラウザが自動的に更新されることを確認
```

### チェックリスト

## トラブルシューティング

### 開発サーバーが起動しない

```bash
# node_modules を再インストール
rm -rf node_modules package-lock.json
npm install
```

### TypeScript エラー

```bash
# tsconfig.json の設定を確認
# 型定義パッケージが不足している場合はインストール
npm install -D @types/xxx
```

### ホットリロードが動作しない

```bash
# vite.config.ts の watch 設定を確認
server: {
  watch: {
    usePolling: true  # Docker 環境では必要
  }
}
```
