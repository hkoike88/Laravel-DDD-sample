# Architecture Decision Records (ADR)

## 概要

本ディレクトリには、プロジェクトにおける重要なアーキテクチャ決定を記録する。
ADR は意思決定の背景・理由・結果を文書化し、将来の参照や見直しに役立てる。

## ADR フォーマット

新しい ADR を作成する際は、以下のテンプレートを使用する。

```markdown
# ADR-XXXX: タイトル

## ステータス

採用 / 却下 / 廃止 / 検討中

## コンテキスト

決定が必要になった背景・課題を記述する。

## 決定

採用する技術・アプローチとその理由を記述する。

## 比較検討

検討した選択肢とその評価を記述する。

## 結果

この決定によって生じる影響（メリット・デメリット・リスク）を記述する。

## 参考資料

関連するドキュメント・URL を記載する。
```

---

## ステータスの定義

| ステータス | 説明 |
|-----------|------|
| 検討中 | 議論中、まだ決定していない |
| 採用 | 決定し、現在有効 |
| 廃止 | 別の決定に置き換えられた |
| 却下 | 検討の結果、採用しなかった |
