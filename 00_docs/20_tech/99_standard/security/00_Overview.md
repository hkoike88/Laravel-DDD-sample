# セキュリティ標準ドキュメント 概要

## 目的

本ディレクトリには、プロジェクト全体におけるセキュリティポリシーと標準を定めたドキュメントを格納する。
開発者・運用者はこれらの標準に従い、安全なシステムの構築・運用を行う。

## 各ドキュメントの概要

### 01_PasswordPolicy.md - パスワードポリシー

NIST SP 800-63B および OWASP に準拠したパスワード管理ポリシーを定める。パスワード構成要件（12文字以上、複雑性）、変更ポリシー、履歴管理、アカウントロックアウト、実装ガイドラインを記載。

### 02_SessionManagement.md - セッション管理ポリシー

セッションのセキュリティポリシーを定める。セッションライフサイクル、ID管理、タイムアウト設定（アイドル30分、絶対8時間）、同時ログイン制御、セッション固定攻撃対策、Remember Me機能の設計を記載。

### 03_DataClassification.md - データ分類・保護ポリシー

システムで取り扱うデータの分類基準と保護要件を定める。4段階の分類レベル（L1極秘〜L4公開）、個人情報（PII）の取り扱い、データライフサイクル管理、アクセス制御、データ転送・共有ルールを記載。

### 04_EncryptionPolicy.md - 暗号化ポリシー

データ暗号化の設計標準を定める。通信暗号化（TLS 1.2以上）、保存データの暗号化（AES-256-GCM）、パスワードハッシュ（Argon2id/bcrypt）、鍵管理、証明書管理、暗号アルゴリズム選定基準を記載。

### 05_IncidentResponse.md - インシデント対応手順

セキュリティインシデント発生時の対応手順を定める。インシデントの分類と重大度（P1-P4）、対応体制、対応フロー（検知→初動→調査→封じ込め→根絶→復旧→事後分析）、証拠保全、ポストモーテムを記載。

### 06_VulnerabilityManagement.md - 脆弱性管理プロセス

脆弱性の検出、評価、対応、修正のプロセスを定める。スキャンスケジュール（CI/CD毎、週次、年次等）、CVSS評価、対応SLA（P1: 24時間以内修正）、修正ワークフロー、依存関係管理を記載。

### 07_ThirdPartySecurity.md - サードパーティセキュリティ

サードパーティ依存関係および外部サービス連携のセキュリティ管理ポリシーを定める。依存関係の脆弱性管理、新規パッケージ導入基準、外部API連携のセキュリティ、ベンダー評価、OSS利用ポリシーを記載。

### 08_SecurityScanning.md - セキュリティスキャンガイド

セキュリティスキャンツールとその使用方法を定める。SAST（静的解析：PHPStan、npm audit）、DAST（動的解析：OWASP ZAP、OpenVAS）、Burp Suite によるペネトレーションテストの実施手順を記載。
