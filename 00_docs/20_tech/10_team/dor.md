# 準備完了の定義（Definition of Ready）

<!--
作業着手前に満たすべき基準
この基準を満たさないとスプリントに入れません
-->

最終更新: 2024-04-01

## 概要

このドキュメントは、ユーザーストーリー・タスクが「着手可能」と言える状態を定義します。
DoR を満たさないストーリーは、スプリントに含めることができません。

## DoR チェックリスト

### 1. 要件の明確性

- [ ] **ユーザーストーリーが明確**
  - As a [ユーザー], I want [機能], So that [価値] 形式で記載
  - 誰が、何を、なぜ必要とするかが明確

- [ ] **受入基準（Acceptance Criteria）が定義済み**
  - 具体的な条件が箇条書きで記載
  - 正常系・異常系の両方を含む
  - 測定可能で検証可能

### 2. 見積もり

- [ ] **チームで見積もり完了**
  - プランニングポーカー等で見積もり実施
  - ストーリーポイントが設定済み

- [ ] **適切なサイズに分割済み**
  - 1スプリント（2週間）内で完了可能なサイズ
  - 目安: 5ポイント以下

### 3. 依存関係

- [ ] **ブロッカーが存在しない**
  - 着手を妨げる障害が解消されている
  - 前提となる機能・環境が準備済み

- [ ] **必要な情報が揃っている**
  - 業務ルールが確認済み
  - 不明点が解消されている

### 4. 設計

- [ ] **UI/画面イメージが共有済み**（UI変更を伴う場合）
  - ワイヤーフレームまたはモックアップが準備済み
  - PO・業務エキスパートが確認済み

### 5. 承認

- [ ] **PO が優先順位を決定済み**
  - バックログ内での順位が明確

---

## MVP フェーズ用 簡易 DoR

Sprint 1〜5（MVP開発期間）は、以下の簡易版を適用します。

### 必須チェック項目

| # | 項目 | 確認者 |
|---|------|-------|
| 1 | ストーリーの目的が明確 | PO |
| 2 | 受入基準が定義済み | PO |
| 3 | 見積もり完了（5pt以下） | チーム |
| 4 | ブロッカーなし | SM |
| 5 | 画面イメージあり（UI変更時） | デザイナー |

### ストーリーテンプレート

```markdown
## ユーザーストーリー

**As a** [ユーザー種別]
**I want** [機能・操作]
**So that** [達成したい価値]

## 受入基準

1. [ ] 〜ができること
2. [ ] 〜が表示されること
3. [ ] エラー時は〜と表示されること

## 見積もり

- ストーリーポイント: X pt

## 備考

- 業務ルール: BR-XXX 参照
- 画面イメージ: [Figma リンク]
```

## ストーリータイプ別の DoR

### ユーザーストーリー（機能開発）

上記の簡易 DoR をすべて満たす

### バグ修正

| # | 項目 |
|---|------|
| 1 | 再現手順が明確 |
| 2 | 期待される動作が明確 |
| 3 | 優先度が設定済み |

### スパイク（調査タスク）

| # | 項目 |
|---|------|
| 1 | 調査目的が明確 |
| 2 | タイムボックスが設定済み（例: 1日） |
| 3 | アウトプット形式が明確 |

## リファインメントの流れ

```
1. PO がストーリー案を作成
   ↓
2. チームで内容を確認・質問
   ↓
3. 受入基準を具体化
   ↓
4. 見積もり実施（プランニングポーカー）
   ↓
5. DoR チェック
   ↓
6. Ready ラベル付与 → スプリント候補へ
```

### リファインメント会議

- **頻度**: 週1回（木曜 15:00〜16:00）
- **参加者**: PO、SM、開発チーム、業務エキスパート（必要時）
- **目的**: 次スプリント分のストーリーを Ready にする

## よくある質問

**Q: DoR を満たさないストーリーを緊急でスプリントに入れたい場合は？**

A: PO とチームが合意すれば例外として認められますが、リスクを明記してください。
未確定部分は「スパイク」として切り出すことも検討。

**Q: 見積もりが5ptを超える場合は？**

A: ストーリーを分割してください。分割の観点：
- 画面ごと
- 機能のバリエーションごと（正常系/異常系）
- ユーザー種別ごと
