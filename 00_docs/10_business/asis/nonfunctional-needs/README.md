# 業務側の非機能ニーズ（AS-IS）

<!--
現行業務から見た非機能要件への期待・課題を整理します
システム化に向けた非機能要件定義のインプットとなります
-->

最終更新: 2024-04-01

## 目的

このドキュメントは、現行の図書館業務から見た **非機能面での期待・課題** を整理します。

- 現行のExcel運用で困っていること（非機能面）
- 新システムへの期待値
- システム非機能要件定義のインプット

**注意**: これは業務側の「期待・ニーズ」であり、システム要件ではありません。
システム要件は別途定義します。

---

## 非機能ニーズのカテゴリ

| カテゴリ | 説明 |
|---------|------|
| 性能 | 処理速度、レスポンス時間への期待 |
| 可用性 | システムを使いたい時間帯 |
| セキュリティ | 情報保護、アクセス制御への期待 |
| ユーザビリティ | 使いやすさ、習得のしやすさ |
| 保守性 | 運用の手間削減 |

---

## 1. 性能

### 1.1 現行の課題

| 業務 | 現行の所要時間 | 課題 |
|-----|--------------|------|
| 貸出上限確認 | 1〜3分 | Excelをフィルタしてカウントするため時間がかかる |
| 返却処理 | 2〜5分 | 貸出管理Excelから該当レコードを探すのに時間がかかる |
| 予約確認 | 1〜2分 | ホワイトボードを見て探す |
| 蔵書検索 | 5〜10分 | 蔵書管理Excelで検索、ファイルが重い |
| 延滞者確認 | 不可能 | リストがないため把握できていない |

**情報源**: 山田さん、佐藤さんへのヒアリング

### 1.2 新システムへの期待

| 業務 | 期待するレスポンス | 理由 |
|-----|------------------|------|
| 貸出処理全体 | **10秒以内** | 利用者を待たせたくない |
| 返却処理全体 | **10秒以内** | 利用者を待たせたくない |
| 蔵書検索 | **3秒以内** | 問い合わせ対応中に確認したい |
| 予約確認 | **3秒以内** | 電話対応中に確認したい |
| レポート出力 | **1分以内** | 月次報告用、多少の待ちは許容 |

### 1.3 同時利用者数

| 項目 | 数値 | 備考 |
|-----|------|------|
| 職員数 | 5名 | 館長1名、司書4名 |
| 同時利用 | 3〜4名 | 全員が同時に操作することは稀 |
| ピーク時 | 5名 | イベント時等 |
| 将来想定 | 10名程度 | 分館設置の可能性あり |

### 1.4 データ量

| データ | 現行件数 | 年間増加 | 備考 |
|-------|---------|---------|------|
| 利用者数 | 約3,000人 | 200人/年 | 有効登録者数 |
| 蔵書数 | 約30,000冊 | 1,000冊/年 | 除籍含まず |
| 貸出件数 | 約50,000件/年 | - | 年間貸出回数 |
| 予約件数 | 約2,000件/年 | - | 年間予約回数 |

**情報源**: 高橋副館長への確認

## 2. 可用性

### 2.1 図書館の開館時間

| 曜日 | 開館時間 | 備考 |
|-----|---------|------|
| 火〜金 | 9:00 〜 19:00 | 通常営業 |
| 土・日 | 9:00 〜 17:00 | 短縮営業 |
| 月曜 | 休館 | 定休日 |
| 祝日 | 休館 | 年末年始を除く |

### 2.2 システム稼働への期待

| 項目 | 期待 | 理由 |
|-----|------|------|
| 開館時間中 | **常に利用可能** | 業務が完全停止するため |
| 閉館後 | 停止可 | 翌日開館前に復旧すればOK |
| 休館日 | 停止可 | 業務なし |

### 2.3 ダウンタイム許容度

| シーン | 許容度 | 影響 |
|-------|-------|------|
| 開館時間中の障害 | **30分以内に復旧** | 貸出・返却が停止 |
| 計画メンテナンス | **閉館後のみ** | 開館中は不可 |

### 2.4 バックアップへの期待

| 項目 | 期待 | 理由 |
|-----|------|------|
| データ消失 | **当日分までは許容** | 翌日に再入力で対応可能 |
| 復旧時間 | **翌日開館までに** | 業務継続のため |

**現場の声**: 「今は紙なので壊れる心配はないが、システムになったら心配」（山田さん）

## 3. セキュリティ

### 3.1 守るべき情報

| 情報種別 | 重要度 | 現行の管理方法 |
|---------|-------|--------------|
| 利用者の氏名・住所・電話番号 | **高**（個人情報） | 利用者台帳（施錠保管） |
| 貸出履歴 | **高**（プライバシー） | 貸出台帳（施錠保管） |
| 予約情報 | 中 | ホワイトボード（誰でも見える） |
| 蔵書情報 | 低 | 蔵書台帳（公開情報） |

**課題**: 予約情報がホワイトボードで丸見え。プライバシー上の問題あり。

### 3.2 アクセス制御への期待

| 操作 | 誰ができるべきか | 備考 |
|-----|----------------|------|
| 貸出・返却 | 全職員 | カウンター業務 |
| 利用者登録 | 全職員 | カウンター業務 |
| 利用者情報閲覧 | 全職員 | 業務に必要 |
| 蔵書登録・除籍 | 担当者のみ | 山田さんが担当 |
| 統計レポート | 館長・副館長 | 報告業務 |
| マスタ設定変更 | 館長のみ | 業務ルール変更 |

**情報源**: 高橋副館長へのヒアリング

### 3.3 監査への対応

| 項目 | 現行 | 期待 |
|-----|------|------|
| 操作ログ | なし | 誰が何をしたかを記録 |
| データ変更履歴 | なし | いつ変更されたかを追跡 |
| ログ保管期間 | - | 3年間（自治体の規定） |

## 4. ユーザビリティ

### 4.1 利用者の特性

| 属性 | 状況 |
|-----|------|
| 年齢層 | 20代〜60代（職員） |
| ITリテラシー | 低〜中（Excel程度は使える） |
| PC経験 | 山田さんは苦手、若手は問題なし |

### 4.2 使いやすさへの期待

| 項目 | 期待 | 理由 |
|-----|------|------|
| 習得時間 | **1日以内** | 業務を止められない |
| 操作ミス防止 | **確認画面必須** | 間違いが怖い |
| マニュアル | **画面付きで必要** | 見ながら操作したい |
| 文字サイズ | **大きめ** | 山田さんは老眼 |

**現場の声**:
- 「パソコンは苦手だから、シンプルにしてほしい」（山田さん）
- 「今のスマホみたいに直感的に使えるといい」（佐藤さん）

### 4.3 入力の効率化

| 現行の課題 | 期待 |
|-----------|------|
| 利用者番号を毎回手書き | バーコードやカードで読み取り |
| 蔵書番号を毎回手書き | バーコードで読み取り |
| 返却期限を計算してスタンプ | 自動計算 |
| 予約連絡先をメモ | 利用者情報から自動取得 |

---

## 5. 保守性・運用

### 5.1 運用負荷への期待

| 項目 | 現行 | 期待 |
|-----|------|------|
| バックアップ | 不要（紙） | **自動で行われる** |
| 障害対応 | - | **専門家に頼める体制** |
| アップデート | - | **勝手に更新される** |
| 日次運用 | - | **特別な作業なし** |

**現場の声**: 「パソコンのことはわからないので、勝手にやってほしい」（山田さん）

### 5.2 サポート体制への期待

| 項目 | 期待 |
|-----|------|
| 問い合わせ窓口 | 電話で相談できる |
| 対応時間 | 開館時間中 |
| 対応速度 | 当日中に回答 |

## 優先順位

ステークホルダーへのヒアリング結果から整理。

| カテゴリ | 優先度 | 理由 |
|---------|-------|------|
| ユーザビリティ | **最優先** | ITに不慣れな職員が使えなければ意味がない |
| 性能 | **高** | 利用者を待たせない（現行より速く） |
| セキュリティ | **高** | 個人情報を扱うため |
| 可用性 | **中** | 開館時間中に使えればOK |
| 保守性 | **中** | 運用負荷は最小限に |

**情報源**: 高橋副館長、館長へのヒアリング

## 暗黙の期待・懸念

ヒアリングで出てきた、明文化しにくい期待や懸念。

| 内容 | 発言者 | 対応 |
|-----|-------|------|
| 「Excelの方が安心」という声がある | 山田さん | 移行期間を設け、並行運用で慣れてもらう |
| 停電時はどうなるか心配 | 館長 | 停電時の業務継続計画を検討 |
| 今の業務フローを大きく変えたくない | 副館長 | 現行業務に近い画面設計 |
| メールアドレスを持っていない利用者もいる | 中村さん | 電話連絡との併用が必要 |
